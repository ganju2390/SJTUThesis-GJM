\chapter{核心硬件平台研发与嵌入式软件开发}

\section{嵌入式软件框架设计}

本系统的嵌入式软件框架旨在实现水下机器人的高精度控制，并确保复杂设备间状态更新与任务执行的有序协同。系统采用统一的设备基类设计，通过 C++ 派生类与多态机制管理底层外设和高层控制模块，实现可扩展、模块化的调度体系。

\subsection{统一设备调度机制}

系统中所有硬件或逻辑功能模块均派生自统一设备基类 \texttt{DeviceBase}，该基类提供标准接口：

\begin{itemize}
    \item \texttt{Update()}：设备状态刷新函数，用于读取传感器数据或更新内部状态。
    \item \texttt{Handle()}：任务执行函数，用于处理控制逻辑或下发命令。
\end{itemize}

所有设备在构造时注册到全局设备队列，通过多态方式统一调度，实现以下特点：

\begin{enumerate}[label=(\arabic*)]
    \item \textbf{自下而上状态更新}：队列头部的底层设备先执行 \texttt{Update()}，逐级向高层模块刷新状态，确保高层逻辑模块使用的数据是最新的。
    
    \item \textbf{自上而下任务执行}：队列尾部的高层控制模块先执行 \texttt{Handle()}，下发控制指令至中低层设备，形成自上而下的任务执行链。
    
    \item \textbf{分频调度}：每个设备可以独立设置更新与任务执行频率（\texttt{divisionFactor}），支持异步周期轮询和不同频率的设备协同。
    
    \item \textbf{软件工程优势}：新增设备仅需继承基类并实现接口函数，无需修改调度器即可被统一管理，实现模块化、可扩展性和高可维护性。
\end{enumerate}

\subsection{ROV 系统设备实例化示例}

以本研究的 ROV 系统为例，设备实例化与调度逻辑如下：

\begin{itemize}
    \item \textbf{高层控制模块}：水下机器人组合控制实例，包含 PID 控制器、状态估计前馈及扰动观测器。
    \item \textbf{上位机指令解码}：包括指令解码模块及对应的串口缓冲，提供数据输入。
    \item \textbf{底层外设}：推进器、深度传感器、IMU。
\end{itemize}

在运行时：

\begin{itemize}
    \item 底层外设首先执行 \texttt{Update()}，刷新传感器状态和执行器反馈。
    \item 上位机指令解码模块随后更新状态，提供高层控制所需输入。
    \item 高层控制模块执行 \texttt{Handle()}，根据底层设备状态和上位机输入计算控制命令，并下发到底层执行器。
\end{itemize}

该顺序保证了控制决策和指令下发的实时性和一致性，为水下机器人的精确控制提供了稳定的软件基础。

\section{推进器驱动与底层通信协议}

传统水下机器人推进器多数采用 PWM 信号进行驱动。PWM 驱动虽然结构简单、易于实现，但存在若干明显局限：

\begin{enumerate}[label=(\arabic*)]
    \item \textbf{抗干扰能力不足}：PWM 信号为模拟脉宽调制波，容易受到噪声、电磁干扰影响，导致推进器响应不稳定。
    \item \textbf{分辨率有限}：PWM 通常使用 8–12 位分辨率，对推力控制精度有限，难以满足高精度控制需求。
    \item \textbf{控制带宽受限}：PWM 信号频率一般较低（~50–400 Hz），导致推进器响应速度和闭环控制带宽受限。
    \item \textbf{缺乏转速回传}：PWM 不提供电子速度反馈，无法实时获得推进器转速信息，不利于实现闭环推力控制。
\end{enumerate}

针对上述问题，本系统采用 DSHOT 数字通信协议驱动推进器，相较于 PWM 具有以下优势：

\begin{enumerate}[label=(\arabic*)]
    \item \textbf{数字信号抗干扰能力强}：DSHOT 使用数字编码控制信号，传输过程具有 CRC 校验，可有效避免通信错误，提高系统可靠性。
    \item \textbf{高分辨率与精确控制}：DSHOT 协议可实现高精度控制（如 11–16 位），显著提升推进器推力调节精度。
    \item \textbf{高频率更新}：DSHOT 支持高达数千 Hz 的更新速率，提供更高控制带宽，使运动控制响应更加及时和稳定。
    \item \textbf{内置转速反馈}：DSHOT 可支持电子转速回传，实现闭环推力控制，保证运动控制的精度和可靠性。
    \item \textbf{简化硬件接线与同步控制}：单根信号线即可传输命令与反馈信息，减少布线复杂度，并便于多推进器同步控制。
\end{enumerate}

通过采用 DSHOT 协议，系统能够实现高精度、高响应、可靠性强的推进器推力控制，为 ROV 的精确运动和稳定控制提供硬件保障。

\subsection{DSHOT通信协议实现方法}

为了实现水下机器人推进器的高精度闭环控制，本系统在自主嵌入式框架上实现了 DSHOT 协议。实现过程与嵌入式调度机制紧密结合，并充分考虑 STM32F4 的 GPIO DMA传输的物理限制：

\paragraph{(1). DSHOT-PIN 和 DSHOT-PORT 封装}  
每个推进器的 DSHOT 控制通道通过单例类封装为 GPIO-PIN，并注册到调度器队列。多个同一物理端口下的 PIN 封装为 GPIO-PORT，便于统一管理和 DMA 批量发送。

\paragraph{(2). 数字信号生成与 CRC 校验}  
每个 DSHOT-PIN 根据期望油门值生成数字信号，并附加 CRC 校验码。生成的信号随后转换为 GPIO BSRR 寄存器的掩码序列，准备通过 DMA 发送到对应 GPIO 端口。

\paragraph{(3). GPIO-PORT 任务执行}  
在 GPIO-PORT 层，将同一 PORT 下各 PIN 的掩码数组合并，形成统一的 BSRR 掩码数组，通过 \texttt{Handle()} 接口触发发送。由于 STM32F4 的物理限制，只有一个DMA通道支持固定频率数据覆写硬件寄存器，
因此来自不同GPIO分组的DHSOT-PORT无法同时更新油门，必须依次发送油门信号并采样回传转速。这里嵌入式系统会依次对不同 GPIO-PORT 执行油门发送和转速读取，保证每个端口的 DMA 传输顺序正确，避免硬件冲突。

\paragraph{(4). DMA 高速数据搬运与频率控制}  
为了满足 DSHOT 协议的更新频率（例如 DSHOT300 的 300 kHz），DMA 采用时钟触发方式搬运 BSRR 掩码数组到 GPIO 寄存器，实现高精度信号输出。每个 GPIO-PORT 在发送完油门数据后，切换为输入模式采样回传数据。

\paragraph{(5). 转速采样与解码}  
转速回传数据采样遵循奈奎斯特采样定理，以三倍 DSHOT 发送频率采样，保证完整捕获回传脉冲。采样完成后进行 CRC 校验和解码，将电子转速（eRPM）转换为实际 RPM，为闭环推力控制提供精确反馈。

\paragraph{(6). 调度机制集成}  
DSHOT-PIN 和 DSHOT-PORT 的构造顺序与嵌入式调度器相结合：先构造 PIN，再构造 PORT，保证油门更新和转速采集能在调度周期内实时执行。Update()/Handle() 接口保证状态更新和任务执行顺序一致，使系统能在高频闭环条件下稳定运行。

\subsection{DSHOT 转速回传验证实验}

为验证 DSHOT 协议实现的闭环推力控制能力，本研究设计了一项转速回传验证实验。实验目标是确认通过 DSHOT 协议获取的电机转速与真实转速（通过无接触式转速计测量）之间的一致性，为后续推力-转速拟合提供可靠数据。

\paragraph{实验方法}  
\begin{enumerate}[label=(\arabic*)]
    \item 将电机固定在实验台上，通过 DSHOT 通信接口下发一系列不同油门值命令。  
    \item 同时使用高精度无接触转速计测量电机的真实转速作为参考。  
    \item 通过嵌入式框架接收 DSHOT 转速回传数据，并记录每个油门值对应的回传转速。  
    \item 对比回传转速与无接触测得真实转速，计算误差、线性拟合系数及 RMSE。
\end{enumerate}

\section{智能电源监控与保护系统设计}

水下机器人在作业过程中，电源系统的安全性直接关系到整机可靠性和电池寿命。高容量电机驱动器、电源滤波电容及电子负载在通电瞬间可能产生大电流冲击，若不采取保护措施，可能导致 MOSFET 或电源板器件损坏、电池过放或过充，
甚至系统重启异常，影响任务执行。为保障系统电源安全，本设计采用 LM5069 热插拔控制器，实现缓启动、欠压/过压保护以及过流与功率限制，从而保护机器人、驱动器及电池的安全。

整个电源保护板主要由缓启动、欠压锁定（UVLO）、过压锁定（OVLO）、过流及功率限制模块构成。缓启动模块通过 TIMER 引脚接入外部电容 C\textsubscript{TIMER}（330 nF），在 LM5069 内部电流源作用下缓慢充电，
使 MOSFET 栅极电压逐步上升，从而限制上电瞬间的浪涌电流，减轻电机驱动器和滤波电容的冲击。欠压和过压保护通过外部分压电阻实现。过流保护模块通过电阻采样负载电流，并结合 MOSFET 并联设计限制电流。

\subsection{缓启动设计与实验}
缓启动通过 TIMER 引脚外接电容 C\textsubscript{TIMER} 实现，其充放电公式如下：
\[
t_\mathrm{inrush} = \frac{C_T \cdot (V_\mathrm{TMRH} - V_\mathrm{TMRL})}{I_\mathrm{charge}} = \frac{330~\mathrm{nF} \cdot 2.75~\mathrm{V}}{5.5~\mu\mathrm{A}} \approx 0.165~\mathrm{s}
\]
实验中，在 24V 电源下负载为电机驱动器和大容量滤波电容，电源开关打开后输出电压呈阶梯型上升，从 0V 到 24V 缓启动耗时约 200 ms，验证了缓启动设计能够有效抑制开机瞬态电流，保护电池和负载（图~\ref{fig:inrush_exp}）。

\begin{figure}[htbp]

    \label{fig:inrush_exp}
\end{figure}

\subsection{欠压与过压保护设计及实验}
通过外部分压电阻设置 UVLO 和 OVLO 阈值：
\[
V_\mathrm{UVLO} = 2.5 \left(1 + \frac{R3}{R4}\right) \approx 18~\mathrm{V}, \quad
V_\mathrm{OVLO} = 2.5 \left(1 + \frac{R6}{R5}\right) \approx 32.5~\mathrm{V}
\]
在 18~V 到 30~V 区间内，以 2V 步进测试输出状态，当电压低于 18V 或高于 30V 时，保护板自动断开输出，输出恢复至安全区间后自动恢复供电，验证了欠压/过压保护的有效性（图~\ref{fig:uvlo_ovlo_exp}）。

\section{本章小结}

本章详细阐述了水下机器人 ROV 控制系统的核心硬件平台与嵌入式软件架构实现。首先，基于 STM32F4 微控制器构建的模块化软件框架采用统一的设备调度机制，
通过 C++ 多态与分频调度实现了底层设备状态的自下而上更新与高层控制指令的自上而下执行，确保了系统的模块化与可扩展性；其次，为克服传统 PWM 驱动的局限，
本系统实现了 DSHOT 数字通信协议，通过 GPIO-DMA 硬件加速、CRC 校验与转速回传验证，实现了高分辨率（11--16 位）、高频率（数千 Hz）的推进器精确控制；
最后，基于 LM5069 芯片的电源保护系统通过缓启动、欠压/过压保护与过流限制等多重防护机制，确保了 ROV 系统在复杂水下环境中的电源安全与长期可靠性。
这些工程实现为第三章提出的控制算法提供了坚实的硬件基础，并为后续实机实验与应用验证奠定了基础。
